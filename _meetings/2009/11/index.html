Topic

RubyGems - Past, Present, and Future

RubyGems is the standard for distributing and managing Ruby code packages. Beginning with Ruby 1.9, RubyGems is included with Ruby itself.

Everyone who has worked with with Ruby is probably already familiar with using RubyGems to download and install Ruby libraries written by others. Gems exist for many tasks, including, for example, manipulating images, validating credit cards, parsing HTML, and generating PDF's. Gems are also an excellent way of distributing your own code if you create a Ruby library you want to share with others.

In this talk, we will briefly discuss the history of RubyGems before focusing on the present and future of RubyGems. Specifically, we will discuss:

  • Using RubyGems and managing your RubyGems environment
  • The transition to GemCutter from RubyForge and Github
  • Creating and publishing your own gems using Jeweler
  • Bundler (a gem to bundle gems)

There will be plenty of time for discussion, so bring your RubyGems questions and ideas.
