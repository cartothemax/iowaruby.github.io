Topic

CoffeeScript

From coffeescript.org…

The golden rule of CoffeeScript is: "It's just JavaScript". The code compiles one-to-one into the equivalent JS, and there is no interpretation at runtime.
  • Why - If it's just JavaScript then why would I want to use it?
  • What - Covering syntax and idioms.
  • Where - I'll be discussing rails integration and how you'd use it outside of Rails.
  • Warts - It's not all rainbows and ponies…I'll show some things that I don't like.
  • Testing - not specific to CoffeeScript, but I'll show how I've been doing headless testing with Jasmine and PhantomJS.
